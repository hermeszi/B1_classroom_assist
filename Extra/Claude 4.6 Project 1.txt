## 🏗️ Recommended Tech Stack (2-Hour Prototype)

| Layer | Choice | Why |
|-------|--------|-----|
| **Frontend** | Single HTML page + vanilla JS | Zero build tools, fast to prototype, still counts as a web app |
| **Backend** | Python **Flask** | Minimal boilerplate, fast API routes, you already know Python |
| **Database** | **SQLite** (file-based) | Zero setup, SQL-capable, easy to seed with test data |
| **AI** | **OpenAI API** (GPT-4o-mini) or **Claude API** | Report generation from raw notes |
| **Voice Input** | Browser **Web Speech API** | Free, built into Chrome/Edge, no server-side setup |
| **Email** | Preview/copy mode (no actual send) | For prototype; show the email draft, don't need Gmail integration |

<details>
<summary><strong>Why not Google Sheets / Tkinter / CSV?</strong></summary>

| Option | Problem |
|--------|---------|
| **Google Sheets as DB** | Requires OAuth2 setup, service account keys, API quotas — burns 30+ min of your 2 hours on config. Save for v2. |
| **CSV** | No concurrent access, no relational queries, messy for multi-student data. Fine for export, bad as primary store. |
| **Tkinter** | Not a web app — B1 explicitly requires "web apps with frontend and backend components." Disqualified. |
| **Local JS only** | No backend = doesn't meet B1 requirement. You need server-side routes. |

**SQLite is the sweet spot**: zero config, relational, and you can always add a "Export to CSV" or "Sync to Google Sheet" button later to show extensibility.

</details>

## 🗄️ Data Model

```sql
-- students table
CREATE TABLE students (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    age INTEGER,
    level TEXT,              -- e.g., "Beginner", "Intermediate", "Advanced"
    parent_email TEXT,
    skills_acquired TEXT,    -- JSON array: ["loops", "functions", "OOP"]
    projects_done TEXT,      -- JSON array: ["Calculator", "Snake Game"]
    notes TEXT               -- general notes about learning style, etc.
);

-- instructors table
CREATE TABLE instructors (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL
);

-- lesson_entries table (core table)
CREATE TABLE lesson_entries (
    id INTEGER PRIMARY KEY,
    student_id INTEGER REFERENCES students(id),
    instructor_id INTEGER REFERENCES instructors(id),
    date TEXT DEFAULT (date('now')),
    raw_notes TEXT,              -- instructor's quick input
    generated_report TEXT,       -- AI-generated lesson summary
    parent_email_draft TEXT,     -- AI-generated email
    handover_notes TEXT,         -- AI-generated for next instructor
    skills_covered TEXT,         -- JSON array
    difficulty TEXT,             -- "easy" / "moderate" / "challenging"
    project_name TEXT,
    suggested_next TEXT,         -- AI suggestion
    created_at TEXT DEFAULT (datetime('now'))
);
```

## 🧠 AI Prompt Design (Key to the Product)

This is the **system prompt** that makes the AI output structured and useful:

```python
SYSTEM_PROMPT = """You are a teaching assistant for a coding/enrichment center. 
Given an instructor's raw lesson notes about a student, generate the following:

CONTEXT PROVIDED:
- Student name, age, level, past projects, skills acquired
- Instructor's raw notes from today's lesson

OUTPUT (return as JSON):
{
  "lesson_report": "2-3 paragraph summary of what happened in class, 
                     what the student worked on, how they performed, 
                     and any observations.",
  "parent_email": "Friendly email to parent summarizing the lesson. 
                    Tone: warm, encouraging, specific. 
                    Mention what child did, what they learned, 
                    and what's coming next. Keep under 150 words.",
  "handover_notes": "Bullet points for the next instructor: 
                      where the student left off, what to continue, 
                      any issues or things to watch for.",
  "skills_covered": ["skill1", "skill2"],
  "difficulty_rating": "easy|moderate|challenging",
  "suggested_next": "Recommendation for next project or activity 
                      based on student's level and progress."
}

Be specific, not generic. Reference the actual work described in the notes.
"""
```

## 🔑 Key Decisions to Document (for `docs/ai-dev/README.md`)

| Decision Point | Options Considered | Choice Made | Rationale |
|---|---|---|---|
| Database | Google Sheets, CSV, SQLite, PostgreSQL | SQLite | Zero config, relational, sufficient for prototype. Google Sheets can be a v2 export target. |
| Frontend framework | React, Vue, Vanilla JS | Vanilla JS + Jinja2 | 2-hour constraint; no build step needed |
| AI model | GPT-4o, GPT-4o-mini, Claude | GPT-4o-mini | Cheap, fast, good enough for structured text generation |
| Voice input | Whisper API, Web Speech API | Web Speech API | Free, client-side, no extra API key needed |
| Email delivery | Gmail API, SendGrid, Copy-to-clipboard | Copy-to-clipboard | Avoids OAuth complexity for prototype |
