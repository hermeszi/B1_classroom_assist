### High-Level Assessment & Strategy

<details>
<summary><b>💡 Why this project is safe, high-impact, and fits the B1 criteria (Click to expand)</b></summary>

1. **High ROI Real-World Problem:** Instructors pacing a class of $2 \le N \le 7$ students spend up to $8 \text{ minutes}$ writing notes per student. For a class of $7$, that’s up to $56 \text{ minutes}$ of post-class admin work. Your tool can bring this down to $\le 1 \text{ minute}$ per student.
2. **True Full-Stack Requirement Accomplished:** Using a backend (to securely communicate with LLM APIs and handle data persistence) and a frontend (for instructor input) demonstrates proper architectural separation.
3. **Strategic Pivot on Database:** Setting up Google Sheets API credential authorization (OAuth/Service Accounts) can easily eat up $45\text{ minutes}$ of your precious $2\text{ hours}$ due to cloud permissions. Instead, use **SQLite** (built directly into Python, zero installation required) as a local staging database, but design the code with a modular repository pattern to prove "Google Sheets / PostgreSQL readiness."
4. **Wow Factor (Voice-to-Text):** Instead of processing raw audio files on the backend (which requires file upload APIs and whisper SDKs), utilize the **HTML5 Web Speech API** in the frontend. It transcribes voice in real-time in the browser for *free* using only ~15 lines of Javascript code!
</details>

### Suggested Technology Stack (Rapid 2-Hour Build)
To hit the strict $T = 2\text{ hours}$ constraint, you need a stack that requires zero compilation, easy setup, and seamless FastAPI integration:

* **Backend:** **FastAPI (Python)**. It is incredibly fast to write, self-documents through Swagger UI, and handles JSON payloads instantly.
* **Frontend:** **Vanilla HTML5 + Tailwind CSS (via CDN) + CSS Grid/Flexbox**. This requires zero build pipelines (no NPM/Vite/Webpack to configure), making running/inspecting the project trivial for the evaluators.
* **Database:** **SQLite**. SQLite stores data in a simple local file (`session_records.db`), requiring no database installation or server setup.
* **AI Integration:** **OpenAI API (`gpt-4o-mini`)** or **Gemini API (`gemini-1.5-flash`)**. (Highly recommend `gpt-4o-mini` or `gemini-1.5-flash` for super-fast latency and incredibly cheap API costs). We will command the model to output a structured JSON schema.

#### 2. The Core Backend API (20 to 60 mins)
Build the FastAPI endpoints in `src/main.py`. Keep it simple:
* `GET /api/students`: Returns a list of current students from SQLite.
* `POST /api/sessions/generate`: Takes student's ID, class context, and raw notes. Calls `ai_helper.py` to analyze the raw notes and returns formatted JSON data (internal logs, parent email, skill ratings).
* `POST /api/sessions/save`: Commits the final generated/edited entry to SQLite database.

```python
# Example Prompt template inside ai_helper.py
SYSTEM_PROMPT = """
You are an expert educational counselor and instructor helper.
Analyze the teacher's chaotic raw class notes and organize them into standardized structured JSON.
Return JSON ONLY matching this structure:
{
  "parent_email_draft": "Warm, encouraging message focusing on progress",
  "internal_handover": "Technical review of specific challenges, code written, and pedagogical recommendations",
  "suggested_next_project": "Name of highly relevant target project based on observed skill gaps",
  "skill_ratings": {
     "logical_thinking": 1-5,
     "code_syntax": 1-5,
     "engagement": 1-5
  }
}
"""
```

#### 3. Frontend Development & Voice-to-Text (60 to 90 mins)

<details>
<summary>🎙️ <b>How to implement Web Speech API in vanilla JS (Click to expand)</b></summary>

Use this clean, native code snippet in your `app.js` to enable immediate voice dictation without third-party libraries:

```javascript
const recognition = new (window.SpeechRecognition || window.webkitSpeechRecognition)();
recognition.continuous = false;
recognition.lang = 'en-US';

document.getElementById('start-record-btn').onclick = () => {
    recognition.start();
    console.log("Listening...");
};

recognition.onresult = (event) => {
    const transcript = event.results[0][0].transcript;
    document.getElementById('raw-notes-input').value += " " + transcript;
};
```
</details>

### Testing & Verification Protocol

To confidently demonstrate the reliability of your prototype in your **B1 Application Presentation and Video (Step 2 & 3)**, execute and document these three testing methods.

#### 1. Automated API Testing (The Technical Verification)
Create a lightweight unit test suite using `pytest` to verify endpoints behave correctly under normal and edge conditions. Run this in your terminal:

```python
# tests/test_api.py
import pytest
from fastapi.testclient import TestClient
from src.main import app

client = TestClient(app)

def test_get_students():
    response = client.get("/api/students")
    assert response.status_code == 200
    assert isinstance(response.json(), list)

def test_generate_session_invalid_student():
    # Pass a non-existent student ID to verify error containment
    response = client.post("/api/sessions/generate", json={
        "student_id": 9999,
        "raw_notes": "Did decent today"
    })
    assert response.status_code == 404
```

#### 2. Manual User Journey Testing (The UI Verification)
Step through this specific flow to document screenshots/GIFs for your `README.md`:
1. **Load Page:** Run `uvicorn src.main:app --reload` and load `http://127.0.0.1:8000`. Confirm database pulls 3 default mock students correctly into the dropdown selection.
2. **Input Test (Short Speech/Text):** Select a mock student (e.g., "Alex, age 11"). Dictate or type a messy, real-world note:
   > *"Alex did loops today. Had some syntax issues with indentation first, but caught on fast. Made a bouncing ball game. Got distracted by playing other games on the web for 10 mins. Ready for simple conditions next time."*
3. **Trigger Generation:** Click "Generate Report". Ensure loading spinners activate properly.
4. **Approve & Edit:** Verify that:
   * **Parent Email** highlights the successful bouncing ball game and frames the loop concept nicely. (Friendly tone checking)
   * **Internal Handover** emphasizes the syntax indentation error and attention monitoring needed next class. (Technical checking)
   * **Skill Rating** parses logical logic as $4/5$, syntax behavior as $3/5$, and engagement as $2.5/5$.
5. **Database Commit:** Click "Save Record". Inspect the SQLite file or a simple table component on your UI to verify the entry was written successfully.

#### 3. LLM Validation & Robustness Checks (AI-Safety Verification)
Show the evaluators that you took AI behavior seriously by performing and listing these safety checks in your `docs/ai-dev/README.md`:
* **JSON Integrity Robustness:** Ensure you utilize **structured outputs** or strict json parsers directly in Python. In the event of an API error, wrap a `try/except` block that falls back to standard text formatting if the JSON fails to parse.
* **Profanity / Guardrails Check:** Test your input text box with non-educational input (e.g. *"This student was an absolute nightmare, worst kid ever."*). Confirm that your system's instruction prompt forces the LLM to neutralize the tone safely and professionally for the parent draft, protecting the brand.
