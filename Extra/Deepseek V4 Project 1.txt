## 3. Prototype Plan (2 Hours)

### Tech Stack Recommendation

| Layer | Technology | Why |
|---|---|---|
| **Frontend** | Plain HTML + CSS + vanilla JS (or Alpine.js / htmx) | Fastest to prototype. No build step. |
| **Backend** | Python + Flask (or FastAPI) | Quick API setup, easy OpenAI SDK integration |
| **Database** | SQLite | Zero config, file-based, perfect for prototype. Migrate to Google Sheets API or Supabase later. |
| **AI** | OpenAI API (gpt-4o-mini) or Claude API | Cheap, fast, good at structured text generation |
| **Email** | Mock/console output for prototype; Resend or SendGrid for real |

> **Why not tkinter?** B1 explicitly requires *"web apps that include frontend and backend components."* A local GUI disqualifies it.  
> **Why not Google Sheets as DB?** For the prototype, SQLite is *faster to set up*. You can mention in your README that production would use Google Sheets API (to integrate with existing workflow) or a proper DB. This shows thoughtful architectural reasoning.


### Database Schema (SQLite — Minimal Viable)

```sql
-- Instructors
CREATE TABLE instructors (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT NOT NULL
);

-- Students
CREATE TABLE students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    parent_email TEXT,
    age INTEGER,
    current_level TEXT,
    enrolled_since DATE
);

-- Classes / Sessions
CREATE TABLE classes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date DATE NOT NULL,
    instructor_id INTEGER,
    subject TEXT,
    project_name TEXT,
    FOREIGN KEY (instructor_id) REFERENCES instructors(id)
);

-- Class-Student attendance
CREATE TABLE class_students (
    class_id INTEGER,
    student_id INTEGER,
    attended BOOLEAN DEFAULT 1,
    PRIMARY KEY (class_id, student_id),
    FOREIGN KEY (class_id) REFERENCES classes(id),
    FOREIGN KEY (student_id) REFERENCES students(id)
);

-- AI-Generated Reports
CREATE TABLE reports (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    class_id INTEGER,
    student_id INTEGER,
    instructor_raw_notes TEXT,          -- the 3-5 bullet points
    ai_generated_report TEXT,           -- polished report
    ai_generated_email TEXT,            -- parent-facing email
    skills_observed TEXT,               -- extracted skills
    difficulty_rating INTEGER,          -- 1-5
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (class_id) REFERENCES classes(id),
    FOREIGN KEY (student_id) REFERENCES students(id)
);
```

## 4. AI Prompt Design

This is the core of your project. The prompt structure is what you'll showcase for B1's *"how you prompted the AI"* requirement.

### System Prompt Template

```python
SYSTEM_PROMPT = """You are an assistant for instructors at an enrichment center 
teaching students aged 9-18. Given raw instructor notes about a student's 
performance in a class, you produce:

1. A concise internal report (for other instructors) covering:
   - What was covered today
   - Skills demonstrated
   - Areas of difficulty (rated 1-5)
   - Recommended next steps / project
   
2. A warm, parent-friendly email update covering:
   - What their child worked on today
   - A specific positive observation
   - One area for growth (framed constructively)
   - What's coming next

Student context:
- Name: {student_name}
- Age: {student_age}
- Current Level: {student_level}
- Today's Subject: {subject}
- Project: {project_name}

Format your response as JSON:
{
  "internal_report": "...",
  "parent_email": "...",
  "skills_observed": ["skill1", "skill2"],
  "difficulty_rating": 3,
  "next_project_suggestion": "..."
}
"""
```

This demonstrates **structured prompting** — you're not just asking the AI to "write something," you're engineering the output format for programmatic use. This is exactly what B1 evaluators want to see.

## 9. Quick-Start Prototype Code (Skeleton)

Here's a minimal Flask app to get you started in the first 15 minutes:

```python
# src/app.py
from flask import Flask, render_template, request, jsonify
import sqlite3
import os
from openai import OpenAI

app = Flask(__name__)
client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

# ── Database init ──────────────────────────────────────────────
def init_db():
    conn = sqlite3.connect("data/enrichment.db")
    conn.executescript(open("data/seed.sql").read())
    conn.close()

# ── Routes ─────────────────────────────────────────────────────
@app.route("/")
def index():
    return render_template("index.html")

@app.route("/api/generate-report", methods=["POST"])
def generate_report():
    data = request.json
    raw_notes = data.get("notes", "")
    student_name = data.get("student_name", "")
    student_age = data.get("student_age", "")
    subject = data.get("subject", "")
    
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT.format(
                student_name=student_name,
                student_age=student_age,
                student_level=data.get("level", ""),
                subject=subject,
                project_name=data.get("project", "")
            )},
            {"role": "user", "content": f"Instructor notes:\n{raw_notes}"}
        ],
        response_format={"type": "json_object"}
    )
    
    import json
    result = json.loads(response.choices[0].message.content)
    
    # Save to SQLite
    conn = sqlite3.connect("data/enrichment.db")
    conn.execute(
        "INSERT INTO reports (student_id, instructor_raw_notes, ai_generated_report, ai_generated_email) VALUES (?, ?, ?, ?)",
        (data.get("student_id"), raw_notes, result["internal_report"], result["parent_email"])
    )
    conn.commit()
    conn.close()
    
    return jsonify(result)

if __name__ == "__main__":
    init_db()
    app.run(debug=True)
```
